# /frontenddesign

Use `$frontend-design` for distinctive, intentional frontend design direction.

## Arguments

- `brief`: The UI, page, component, or redesign request to apply the frontend design skill to.

## Workflow

1. Load the `$frontend-design` skill.
2. Ground the design in the specific subject, audience, and job of the requested UI.
3. Create a compact design plan covering palette, type, layout, and one signature element.
4. Critique the plan for generic defaults and revise before building.
5. Implement or advise using the revised plan, then visually review the result when possible.
